Error's I encountered

I made an error while making this function when sending the details from the detail's function to the detail's list in the main function. This issue occured because I appended the detail's list called 'pickup_information' instead of extending it. I used append in the first place because that was the command I was familiar with as it was what I used to change pizza quantities and pirces. Appending is only used to change one part of a list, while extend is used to add all parts of the list. To fix the probelm of using append, I just switched the code to say extend. The details informationw as needed to be sent back to the main function to change the detail's list, but it's also needed to be sent to the print_details function.

Problem I encountered:

A problem that occured was that when canceling the order, it only cancels the order and not the details that are stored. So if I were to cancel a order with details loaded and then start a new order my details would already be there. This is a problem if the next person that tries to order has different details, which is most likely. It would work alright if the customer rememebers to put in details before finalising their order. However, if the user doesn't input their details, the program will think they have their details and finalise their order with the wrong details.

To fix this I decided that the cancel function should also delete the details.

Problem Fixed:



However, upon finding the solution to this problem, I was thinking about the whole idea of the cancel function. At the moment the customer is only allowed to cancel their order if they have something in their order. Then i thought about the possibility of a person who has just filled out their details, and has not ordered yet, and wants to cancel their order. I realised that this may be a bit consfusing to the customer. This is because the conditions on the cancel function wouldn't allow this person to cancel their order.

Reflection:

This sprint focused on getting and storing the customer details. I created two functions, one to get the details and one to print them out. The details that the program got were extended to the detail's list in my main function. This was important so the information could be stored properly.

Another important part of this details function, is making sure that the service charge was correct. To do this I returned 3 if the customer chose the delivery option and I returned 0 if the customer chose the pickup option. Then I had 'service_charge = details(details_list)' which meant I had a variable which I could add to the total cost.

The testing was very interesting as there were so many inputs by the user, and a few problems were created. I fixed most of them to the best of my ability. However, I couldn't fix one problem which was the phone number inputs. I hope to work on this problem in the future when I have more time. This really shows how fragile programs are without proper validation functions in place.

The last necessary feature to add is the finalise the order feature. I think this is necessary to the code as I will be able to give the user a fully summary of what they have entered and give the customer information about how much GST they're paying.
